🐛 PROBLEM:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Lesson cards on homepage were flickering/blinking continuously,
especially when hovering over the card edges.

ROOT CAUSES:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. ❌ Continuous animation on ::before element (rotateBorder)
   └─ Always running, causing visual noise
   
2. ❌ Wiggle animation triggered on hover
   └─ Conflicted with hover state
   └─ Caused rapid on/off toggling on edges
   
3. ❌ Complex border implementation
   └─ 4px transparent border + pseudo-element
   └─ Excessive positioning logic

✅ SOLUTIONS APPLIED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FIX #1: Remove Continuous Animation
────────────────────────────────────
BEFORE: .lesson-card::before { animation: rotateBorder 3s infinite; }
AFTER:  .lesson-card::before { opacity: 0; transition: opacity 0.3s; }
        .lesson-card:hover::before { opacity: 1; }

FIX #2: Simplify Hover Effect
──────────────────────────────
BEFORE: .lesson-card:hover { animation: wiggle 0.5s ease-in-out; }
AFTER:  .lesson-card:hover { 
          transform: translateY(-8px);
          box-shadow: 0 12px 30px rgba(...);
          border-color: #4ECDC4;
          background: linear-gradient(...);
        }

FIX #3: Optimize CSS Transitions
─────────────────────────────────
BEFORE: transition: all 0.3s cubic-bezier(0.68, -0.55, 0.265, 1.55);
AFTER:  transition: all 0.3s ease;
